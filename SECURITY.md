# Security Policy

If you discover a security vulnerability, please report it privately:

Email: vishwanathbarve1@gmail.com
 
## Please include:

Description of the vulnerability , Steps to reproduce , Any potential impact , Your contact info for follow-up

We do not accept vulnerability reports via public issues to protect users.


# Data Security Measures

Encryption: All user data is encrypted at rest (AES-256) and in transit (TLS/HTTPS).

Access control: Role-based access for admin and system accounts only.

Local processing: Users can choose to keep data on their own devices.

No third-party sharing: User data is never sold, rented, or shared without consent.

 # User Recommendations

Use strong, unique passwords for accounts.

Keep devices and apps up-to-date.

Report any suspicious behavior or potential security issues immediately.

# Acknowledgements

We follow industry best practices including HIPAA/GDPR principles where applicable and conduct regular security audits to ensure the safety and privacy of user data.
